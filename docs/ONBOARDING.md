# Onboarding — как принять нового узла в складчину

Workflow для тебя как админа. Описывает что делать **каждый раз когда новый друг хочет войти**.

Готовый шаблон сообщения для отправки другу — в [INVITE_TEMPLATE.md](INVITE_TEMPLATE.md).

---

## Перед первым приглашением — один раз

### 1. Создай Telegram-чат для узлов
- Назови «Sethub VPN» или что-то нейтральное.
- Это **приватный** чат, не канал. Только реальные знакомые.
- Закрепи в чате:
  - Ссылку на CLIENTS.md / `clients.sethub.org`
  - Краткие правила (ниже)
  - Реквизиты для перевода

### 2. Правила (закрепи)
```
🎯 Что это
Личная VPN-инфраструктура, в складчину среди знакомых. Не услуга.

💰 Сколько
~Х₽/мес с узла. Сумма зависит от числа узлов. Считаем раз в месяц.
Платим до Хчисла. Если не платишь — отключаю в начале следующего.

🤝 Правила
- Не делиться ссылками с НЕ-узлами
- Не использовать для торрентов / майнинга / DDoS
- Никакого незаконного контента (детский — мгновенный бан)
- Жалобы / проблемы — в чат, без личек

📲 Установка
clients.sethub.org — там клиенты по всем платформам и инструкции
```

### 3. Заведи табличку (Google Sheets / просто текстовый файл)
| Узел | Tg | Marzban username | Дата старта | След. оплата | Статус |
|---|---|---|---|---|---|
| Вася | @vasya | vasya | 2026-06-15 | 2026-07-15 | active |
| Петя | @petya | petya | 2026-06-20 | 2026-07-20 | pending payment |

Хватит этого до 15-20 узлов. Дальше — Telegram-бот или плагин в Marzban.

## Workflow на каждого нового друга

### Шаг 1. Договорённость
Друг говорит «хочу». Ты отправляешь ему [INVITE_TEMPLATE.md](INVITE_TEMPLATE.md) (целиком в Telegram, как сообщение). Там описано всё что ему нужно знать **до согласия**.

### Шаг 2. Узел соглашается → создаёшь юзера в Marzban
1. Открой Marzban: `https://45.152.22.222:8003/dashboard/` (или `https://vpn.sethub.org/dashboard/`).
2. **Create user**:
   - **Username:** `vasya` (короткое, латиница, для самоидентификации в логах)
   - **Inbounds:** галки на **ВСЕ** три: `VLESS Reality`, `VLESS XHTTP Reality`, `VLESS XHTTP CF`
   - **Expire date:** дата до которой оплачено + 1 день (например, оплатил по 15 июля → `2026-07-16`)
   - **Data limit:** оставь пустым (без лимита для своих)
   - **Note:** `Вася / tg @vasya / оплачено до 2026-07-15`
3. Save → нажми на пользователя → копируй **Subscription URL**.

### Шаг 3. Собери Hysteria2-ссылки руками
Hysteria2 общий для всех (не пер-юзер), но друг может вставить:

**Базовая Hysteria2 (для мобиле где работает UDP):**
```
hysteria2://AUTH_PASSWORD@hy.sethub.org:36712/?obfs=salamander&obfs-password=OBFS_PASSWORD&sni=hy.sethub.org#Hysteria2-DE
```

**Hysteria2 с port hopping (резерв если оператор начнёт точечно блочить 36712):**
```
hysteria2://AUTH_PASSWORD@hy.sethub.org:36712/?obfs=salamander&obfs-password=OBFS_PASSWORD&sni=hy.sethub.org&ports=20000-50000#Hysteria2-DE-Hop
```

⚠ Если в `AUTH_PASSWORD` / `OBFS_PASSWORD` символы `+`, `/`, `=` — заменяй на `%2B`, `%2F`, `%3D` соответственно.

### Шаг 4. Отправь другу пакет
В личку (одним сообщением, чтобы не потерялось):

```
Привет! Вот твой персональный комплект:

🔗 Subscription URL (4 канала в одной ссылке):
<URL из Marzban>

🌐 Hysteria2 (отдельная ссылка):
<сборная ссылка из шага 3>

📲 Установка клиента: https://clients.sethub.org/
   - Hiddify — рекомендую первым делом
   - Подробно по платформам: https://github.com/mikey-semy/clients-mirror/blob/main/docs/CLIENTS.md

🚀 Действия:
1. Установи Hiddify (или Streisand на iPhone)
2. В клиенте → «Add from clipboard» (+ в правом верхнем) → импорт subscription URL
3. Импорт Hysteria2 ссылки — отдельно тем же способом
4. Connect → тест на claude.ai

💸 Переводы — на <реквизиты>
   Сейчас Х₽/мес. Следующая оплата у тебя <дата>.

Если что-то не работает — в чат [ссылка на Sethub VPN чат].
```

### Шаг 5. Запиши в табличку
- Username, Telegram, дата старта, дата следующей оплаты
- Статус: `active` после первого успешного коннекта (он скажет «работает!»)

### Шаг 6. (Опционально) Добавь в Telegram-чат «Sethub VPN»

## Workflow продления подписки

Раз в месяц проверяешь табличку. Кто не заплатил до даты:
1. Личкой: «Эй, у тебя оплата истекла N дней назад, продлеваем?»
2. Если ОК → Marzban → user → меняешь `Expire date` на месяц вперёд.
3. Если молчит / не платит → меняешь expire на «вчера» (или просто `Disable` в Marzban) → юзер автоматически отключается. **Не удаляй**, может ещё захочет вернуться.

## Workflow возврата

Узел молчал 3 месяца, потом «эй, я тут вернулся»:
1. Marzban → user → Enable → новый Expire date.
2. **НЕ нужно** перевыпускать ссылки — UUID не менялся, старая subscription у него ещё работает.

## Workflow увольнения

Если узел нарушает правила (торренты, шарит ссылку наружу):
1. Marzban → user → **Delete**. UUID сжигается.
2. Если просто перестал платить и больше не интересно — оставь Disabled. Это не вредит.

---

## Что не делаешь

- ❌ **Не выдавай subscription URL «гостям»** для теста. Каждый — свой юзер.
- ❌ **Не делай узлам admin-доступ** к Marzban-панели. Только ты.
- ❌ **Не публикуй ссылки** в открытых каналах / на форумах.
- ❌ **Не принимай деньги «за услугу»** — это складчина, разделение издержек. В Telegram пиши «скидываемся на сервер», не «оплата подписки».

## Если узлов становится > 15

- **Telegram-бот** с интеграцией в Marzban API — выдаёт ссылки автоматически по UUID, напоминает об оплате. Кодить ~1 вечер.
- Или **готовое решение** — Marzban-Bot (есть community-боты под Marzban).
- Хороший момент задуматься: это уже больше «сервис», чем «складчина». Подумай юридически.
